# Competitor implications

| Product       | Adopt                                                                  | Avoid in the first release                              |
| ------------- | ---------------------------------------------------------------------- | ------------------------------------------------------- |
| Roll20        | token permissions, chat rolls, character-linked tokens, shared jukebox | macros, mod API, dynamic lighting, multi-system breadth |
| Owlbear Rodeo | fast room entry, scene-first map workflow, simple manual fog           | drawing suite and extension platform                    |
| Quest Portal  | universal sheets and rolls from a sheet                                | AI, voice, native mobile applications                   |
| D&D Beyond    | separate reusable content library                                      | commerce, subscriptions and license entitlements        |
| Foundry VTT   | self-hosting and renderer-independent game documents                   | product/runtime dependency                              |

The MVP should feel closer to a focused Owlbear-style table connected to a Quest Portal-style sheet than a reduced copy of every Roll20 panel.
