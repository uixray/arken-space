## Design Context

### Users

One GM and five to six invited friends use arken-space in desktop browsers during real tabletop sessions. The GM prepares scenes and controls hidden information; players need to enter quickly, read their character state and act without learning a complex tool.

### Brand Personality

Raw, minimal and functional. This phase is a concept and usability test, not a visual-brand exercise.

### Aesthetic Direction

A dense, neutral application shell with the map taking most of the viewport. Use plain surfaces, direct labels, obvious focus and selection states, and minimal motion. Avoid decorative fantasy motifs, gradients, glass effects and ornamental panels. A single restrained dark theme is acceptable for the prototype, but theming and branded visual design are explicitly deferred.

### Design Principles

1. The current game state must be understandable at a glance.
2. The map gets priority over navigation and chrome.
3. One obvious action is better than a flexible but hidden control system.
4. GM and player permissions must be visible as well as enforced.
5. Every element must help test the product concept; decoration waits.
